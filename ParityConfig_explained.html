1. หลักการ — "Model Artifact = Self-Contained Config"

เดิม model carry เฉพาะ weights + norm stats แต่ ไม่ได้บอก ว่า features ถูกคำนวณด้วย periods/thresholds อะไร → EA ต้อง "เดา" (= ใช้ defaults)

ตอนนี้: config.mqh = self-describing artifact

┌──────────────────────────────────────────────────────────────────┐ │ <model>_config.mqh (auto-generated by export_to_onnx.py) │ ├──────────────────────────────────────────────────────────────────┤ │ RL_INPUT_DIM, OUTPUT_DIM, WINDOW_SIZE, FEATURE_COUNT │ │ RL_FEATURE_NAMES[] — รายชื่อ feature ตามลำดับ │ │ RL_FEAT_MEAN[], RL_FEAT_STD[] — norm stats │ │ ⭐ RL_ApplyDataCollectorConfig() — periods + candle params │ ← NEW! │ RSI_PMIN = 4; RSI_PMAX = 30; │ │ ATR_PMIN = 5; ATR_PMAX = 50; │ │ CP_MarubozuThresh = 0.95; ... │ │ (ค่าเดียวกับที่ collector ใช้ตอน gen data — embed อัตโนมัติ) │ └──────────────────────────────────────────────────────────────────┘
ผลลัพธ์: ใครจะ deploy โมเดลนี้ — แค่ #include config.mqh แล้วเรียก RL_ApplyDataCollectorConfig() ก่อน init → features ที่คำนวณตอนรัน จะตรงกับตอน train เสมอ ไม่ว่าผู้ใช้จะปรับ periods แค่ไหนใน collector

params_file = SCRIPT_DIR / f"{model_name}.params.json"
if params_file.exists():
    params = _json.loads(params_file.read_text())
    mqh.append("void RL_ApplyDataCollectorConfig()")
    mqh.append("{")
    for k, v in params.items():
        mqh.append(f"   {k} = {fmt(v)};")
    mqh.append("}")
else:
    mqh.append("void RL_ApplyDataCollectorConfig() { /* no-op */ }")
