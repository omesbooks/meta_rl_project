📥 Data Collection

เก็บข้อมูลตลาดจริงผ่าน MT4 Strategy Tester

📌 What we did

  • เขียน EA สำหรับ MT4 ที่เขียน CSV ทุก bar
  • เก็บ OHLCV + indicators (RSI, EMA, ATR, MACD, BB...)
  • รัน Strategy Tester period 2010-2025 ด้วย tick data
  • คำนวณ future return → label เป็น UP/DOWN/FLAT

🛠️ Tools / Files

  • DataCollector.mq4 — v1 (6 features)
  • DataCollector_v2.mq4 — v2 (35 features) ⭐
  • Output: EURUSD_H1.csv, training_data_v2.csv

📊 Result

  • EURUSD H1: 100,748 rows × 6 features (v1)
  • XAUUSD H1: 96,661 rows × 35 features (v2)
  • Tick data จาก Dukascopy = accurate

💡 Key Lesson

  • คุณภาพข้อมูล = พื้นฐานทุกอย่าง
  • Tick data > broker hist data
  • เก็บ features ตอนแรกเลย ดีกว่ามาทำใน Python
2

🔧 Data Processing

เตรียมข้อมูลให้พร้อมเทรน — แก้ class imbalance + เพิ่ม temporal features

📌 What we did

  • Relabel: แก้ปัญหา FLAT 73% → quantile 33/33/33
  • Add lag features (1, 3, 5, 10, 20 bars history)
  • Rolling stats (mean, std) + diff features

🛠️ Tools / Files

  • relabel.py — quantile/binary/threshold modes
  • add_lag_features.py — temporal context

📊 Result

  • Class balance: FLAT 73% → 33% (ดีกว่ามาก)
  • Features: 6 → 35 → 280 (with lags)

💡 Key Lesson

  • Class imbalance = lazy classifier trap
  • Feature engineering > algorithm tuning
  • แต่ระวัง overfit ถ้า features เยอะเกิน
3

🌲 Supervised Learning (PyCaret)

ลองโมเดล classification — เทรน + คาดเดา UP/DOWN/FLAT

📌 What we did

  • สร้าง GUI app ด้วย Tkinter เทรนได้ง่าย
  • ใช้ PyCaret compare_models หา algo ที่ดีสุด
  • Time-series split + fix imbalance (SMOTE)
  • Walk-forward + confidence analysis

🛠️ Tools / Files

  • app.py — GUI training
  • predict.py, backtest.py
  • analyze_confidence.py

📊 Result (EURUSD H1, 6 features)

  • Win Rate: 45.71% ❌
  • Profit Factor: 0.83 ❌
  • Return: -85% (เพราะ no edge)

💡 Key Lesson

  • Data leakage = ทำลายทุกอย่าง (future_return เป็น feature)
  • EURUSD H1 + 6 features = ไม่มี edge
  • ต้องเปลี่ยน symbol หรือเพิ่ม features
4

🤖 Reinforcement Learning (PPO)

Agent เรียนเทรดเองจาก reward — มี state + action + memory

📌 What we did

  • สร้าง TradingEnv (Gymnasium) — state + action + reward
  • เทรน PPO 200k steps ด้วย stable-baselines3
  • แก้ buy-and-hold trap ด้วย realized reward + max_hold
  • เปรียบเทียบ window size 10 vs 100 (overfit!)

🛠️ Tools / Files

  • trading_env.py — Gym environment
  • rl_train.py — PPO trainer
  • rl_backtest.py, rl_analyze.py
  • rl_backtest_filtered.py — confidence filter

📊 Result (XAUUSD H1, 35 features)

  • Win Rate: 52.41% ✅
  • Profit Factor: 1.02 (1.15 with filter) ✅
  • Return: +1.92% (test set)

💡 Key Lesson

  • Symbol matters! XAUUSD ≠ EURUSD
  • Reward shaping = หัวใจของ RL
  • มากเกินไป = overfit (window=100 พัง)
5

🔬 Validation & Robustness

พิสูจน์ว่า model robust จริงๆ ไม่ใช่ luck

📌 What we did

  • Walk-forward validation 5 windows
  • Train + test ซ้ำหลายช่วงเวลา
  • วิเคราะห์ concept drift
  • วางแผน adaptive retraining strategy

🛠️ Tools / Files

  • rl_walkforward.py — WF validator
  • Output: wf_xau_v3_results.csv
  • Output: wf_xau_v3_chart.png

📊 Walk-Forward Result

  • Window 1-3: PF 1.07-1.28 ✅
  • Window 4-5: PF 0.97-0.86 ❌
  • 3/5 ผ่าน → UNSTABLE

💡 Key Lesson

  • Concept drift = real! (ตลาดเปลี่ยน)
  • Single backtest หลอก, walk-forward เปิดเผยความจริง
  • ต้องมี retrain strategy ทุก N เดือน

🎓 7 บทเรียนสำคัญ

🎯

1. Symbol Matters Most

EURUSD ที่เป็น efficient market สู้ ML/RL ไม่ได้ XAUUSD ที่มี structure มากกว่า → edge เกิดได้ง่ายกว่า

🔧

2. Features > Algorithm

6 features ทำไม่ได้ + 35 features ทำได้ — feature engineering สำคัญกว่าเลือก algorithm

⚠️

3. Data Leakage Kills

future_return เผลอใส่เป็น feature → model accuracy 100% (หลอก) → live = พัง

🎁

4. Reward Shaping = Heart of RL

MTM reward = buy-and-hold trap. Realized reward + max_hold = real trading

📏

5. Goldilocks Principle

น้อยเกินไป (6 feat) underfit. มากเกินไป (window 100) overfit. ต้องหาจุดกลาง

🔬

6. Walk-Forward = Truth Serum

Single backtest หลอกได้. Walk-forward เปิดเผย concept drift ที่ซ่อนอยู่

🔄

7. Edge ไม่ Permanent

ตลาดเปลี่ยน → model เสื่อม → ต้อง retrain ทุก 3-6 เดือน. Deploy แล้วทิ้ง = ผิด

📖 Story การพัฒนา (เรียงตามลำดับ)

1

เริ่มต้น — สร้าง PyCaret GUI Trainer

สร้าง app.py ให้แนบไฟล์ CSV เทรนได้ทันที — รองรับ classification/regression/clustering

2

เก็บข้อมูล MT4 — DataCollector EA

เขียน EA เก็บ OHLC + indicators ลง CSV รันใน Strategy Tester period 2010-2025

3

เจอ Bug ใหญ่ — Data Leakage!

Win rate 100% ดูเทพ แต่จริงๆ future_return ปนเป็น feature → ต้องแก้

4

Retrain แล้วเจอ — Class Imbalance

FLAT 73% → model ขี้เกียจ ทำเฉพาะ FLAT → สร้าง relabel.py แก้

5

EURUSD ไม่เวิร์ค — ลอง RL

PyCaret WR 45% — สร้าง TradingEnv + PPO หวังว่า RL จะดีกว่า

6

RL Buy-and-Hold Trap

Agent เปิด long ครั้งเดียว ถือ 3 ปี! → แก้ด้วย realized reward + max_hold_bars

7

Big Insight — เปลี่ยน Symbol

เปลี่ยนเป็น XAUUSD + เพิ่ม features เป็น 35 → WR 52%, PF 1.02 ✅

8

ลอง Window=100 — Overfit หนัก

คิดว่ามากดี แต่ผลตรงข้าม! เรียนรู้ "Goldilocks Principle"

9

Walk-Forward — เผย Concept Drift

3/5 ผ่าน, ผลของ window 4-5 (ปีล่าสุด) แย่ลง → edge เคยมี แต่กำลังหายไป

10

วางแผน Production — Retraining Strategy

Periodic retrain vs Online fine-tuning vs Continual learning — ต้องมี maintenance plan
