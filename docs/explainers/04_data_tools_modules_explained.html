Data Tools Modules Explained

หน้า Data Tools คือพื้นที่เตรียมข้อมูลก่อนเข้า Train, Backtest และ Export จุดประสงค์คือให้ user ทำงานกับไฟล์จาก MT5 ได้โดยไม่ต้องเปิด command line

หน้า Data Tools ใช้ทำอะไร

ให้คิดหน้านี้เป็นโต๊ะเตรียมวัตถุดิบก่อน train โมเดล RL: เอาไฟล์จาก MT5 เข้ามา, แบ่งข้อมูลตามเวลา, ตรวจ feature ที่ซ้ำกัน, และส่งโมเดลที่ผ่านการ train แล้วออกไปเป็นไฟล์ MT5

1. Import

ดึง CSV จาก MT5 เข้ามาในโปรเจกต์

2. Split

แยกอดีตเป็น train และอนาคตเป็น test

3. Clean

เช็ค feature ซ้ำหรือสัมพันธ์กันสูง

4. Export

แปลงโมเดลเป็น ONNX และไฟล์ MT5

5. Log

ดูผลลัพธ์และ error ของทุกเครื่องมือ

โมดูลในหน้า Data Tools

1
First step

Import from DataCollector_RL

ใช้คัดลอกไฟล์ CSV ที่ DataCollector_RL เขียนไว้ใน MT5 จาก Common\Files เข้ามาไว้ใน project folder พร้อมคัดลอกไฟล์ .params.json ถ้ามี

  • ใช้เมื่อ: เพิ่งรัน DataCollector_RL ใน MT5 tester หรือ live chart เสร็จ
  • Input: Source CSV จาก MT5 Common Files
  • Output: CSV สำหรับ train เช่น training_data_gu_h4_dataset.csv
  • สำคัญ: .params.json ช่วยให้ feature/indicator settings ตรงกันระหว่าง train กับ EA
2
Validation

Split Train/Test by Date

ใช้แยกข้อมูลเป็น 2 ไฟล์ตามวันที่: ฝั่งก่อนวัน split เป็น train และฝั่งตั้งแต่วัน split เป็นต้นไปเป็น test

  • ใช้เมื่อ: อยากมีไฟล์ train/test แยกชัดเจนสำหรับสอนหรือทดสอบ out-of-sample
  • Input: CSV ที่มีคอลัมน์ timestamp
  • Output: ไฟล์ _train.csv และ _test.csv
  • ข้อควรรู้: Train page มี Train % อยู่แล้ว จึงไม่จำเป็นทุกครั้งสำหรับ RL basic flow
3
Optional

Auto-remove Redundant Features

เป็นตัวเลือกในโมดูล Split ถ้าเปิดไว้ ระบบจะลบ feature ที่ correlation สูงเกิน threshold หลัง split แล้วใช้ชุด feature เดียวกันกับ train และ test

  • ใช้เมื่อ: dataset มี feature จำนวนมากและสงสัยว่าหลายตัวซ้ำกัน
  • ค่า default: threshold 0.85
  • ประโยชน์: ลดขนาด input ของ neural network และลดโอกาส overfitting
  • ข้อควรระวัง: ถ้าลบ feature มากเกินไป โมเดลอาจเสีย signal ที่มีประโยชน์
4
Feature QA

Feature Analysis & Cleanup

ใช้ตรวจความสัมพันธ์ระหว่าง features และทำความสะอาด feature ที่ซ้ำกัน โดยไม่จำเป็นต้อง split ก่อน

  • Show Correlation Matrix: ดูภาพรวมว่า feature ไหนสัมพันธ์กันสูง
  • Clean Redundant Features: สร้างไฟล์ใหม่ที่ตัด feature ซ้ำออก
  • Input: CSV ที่เลือกจาก dropdown
  • เหมาะกับ: การเตรียม dataset ก่อน train รอบจริง หรือก่อนสอนเรื่อง feature quality
5
Deploy

Export to MT5 (ONNX)

ใช้หลังจาก train และ backtest จนพอใจแล้ว เพื่อแปลงโมเดล PPO เป็นไฟล์ ONNX และสร้างไฟล์สำหรับ MT5

  • Input: model ที่ train แล้ว เช่น rl_prod_v1.zip หรือ best model
  • Output: Files/*.onnx, Include/*_config.mqh, Experts/*_EA.mq5
  • ใช้เมื่อ: ต้องการเอาโมเดลไปทดสอบหรือ deploy ใน MT5
  • ข้อควรจำ: ไม่ใช่ขั้นตอน data prep แต่เป็นขั้นตอนหลัง train/backtest
6
Debug

Tools Log

เป็นกล่องข้อความที่แสดงสถานะของทุก action ในหน้า Data Tools เช่น import สำเร็จ, split ได้กี่ rows, export ล้มเหลวเพราะไม่เจอ model

  • ใช้เมื่อ: ต้องการรู้ว่าเครื่องมือทำอะไรไปแล้ว
  • ช่วย debug: ถ้ามี error ให้ดูข้อความใน log ก่อน
  • สำหรับสอน user: ให้ user อ่าน log เพื่อเข้าใจว่าขั้นตอนสำเร็จหรือยัง

ควรใช้ลำดับไหนในงานจริง

สถานการณ์ ลำดับที่แนะนำ หมายเหตุ
สอน user ตั้งแต่เริ่มเก็บข้อมูล Import -> Split optional -> Feature Analysis optional -> Train page ถ้าเน้น RL basic ให้ Import แล้วไป Train page ได้เลย เพราะ Train page มี train/test split ตามเปอร์เซ็นต์อยู่แล้ว
เตรียม dataset ให้สะอาดก่อน train รอบจริง Import -> Feature Analysis -> Clean Redundant Features -> Train page เหมาะกับ dataset ที่มี feature เยอะหรือ indicator หลายชุดที่อาจซ้ำกัน
ต้องการ test แบบแยกไฟล์ชัดเจน Import -> Split Train/Test by Date -> Train ด้วย train file -> Backtest ด้วย test file ดีสำหรับสอนเรื่อง out-of-sample และหลีกเลี่ยง look-ahead bias
โมเดลผ่าน backtest แล้ว จะเอาไป MT5 Export to MT5 (ONNX) ใช้หลังจาก model ดีพอแล้ว ไม่ควร export โมเดลที่ยัง backtest แพ้

สิ่งที่ถูกตัดออกจาก RL-only flow

โมดูล Relabel UP/DOWN/FLAT และการ run build_training_from_collector.py ถูกซ่อน/ตัดออกจากหน้าใช้งานหลัก เพราะเป็นแนว supervised ML/PyCaret มากกว่า RL ส่วน RL ใช้ action + reward จาก TradingEnv ไม่ได้ใช้ target label แบบ UP/DOWN/FLAT เป็นคำตอบหลัก

สรุปสั้นสำหรับ user

โมดูล คำจำง่าย
Import from DataCollector_RL เอาข้อมูลจาก MT5 เข้ามาในโปรเจกต์
