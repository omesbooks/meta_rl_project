ระบบ Reward ของโมดูล Train RL

ตอนนี้ระบบ train ใช้ Reward แบบ realized V4 Honest Reward เป็นค่าแนะนำ เป้าหมายคือให้ agent เรียนรู้จากกำไรขาดทุนที่ปิดไม้จริง มากกว่าการไล่คะแนนจากกำไรลอยตัวชั่วคราว

ภาพรวมแบบสั้น

Default

โหมดหลักคือ realized

Train page และ Full Pipeline ส่งค่า --reward_mode realized เข้า rl_train.py เป็นค่าเริ่มต้น

Core

Reward อยู่ใน TradingEnv

ไฟล์หลักคือ trading_env.py โดย env เป็นคนคำนวณ position, P/L, force close และ reward ทุก step

Goal

กัน over-trading

มีค่าปรับตอนเปิดไม้และตอนอยู่เฉย ๆ เพื่อบังคับให้ agent เทรดเมื่อสัญญาณคุ้มจริง

Action ที่ agent เลือกได้

0

Hold

ไม่ทำอะไร ถ้า flat แล้ว hold จะโดนค่าปรับเล็กน้อย

1

Buy

เปิด long ถ้ายังไม่มี position และโดน spread ตอนเข้า

2

Sell

เปิด short ถ้ายังไม่มี position และโดน spread ตอนเข้า

3

Close

ปิด position ปัจจุบัน ตรงนี้คือจุดที่ reward หลักจาก net P/L ถูกให้

สูตร Reward ปัจจุบัน

สูตรนี้ไม่ได้ให้คะแนนจากกำไรลอยตัวเต็ม ๆ แต่ให้เป็นแค่สัญญาณช่วยเล็กน้อย คะแนนใหญ่จริง ๆ จะเกิดเมื่อปิดไม้แล้วรู้ net P/L

reward = unrealized_delta * 0.1 + net_pnl_on_close * 50 + 0.01 if net_pnl_on_close > 0.5% - 0.005 when opening a trade - 0.0001 when holding flat - 0.0002 after holding too long - 0.001 when giving back more than half of a good floating profit

แต่ละส่วนแปลว่าอะไร

เหตุการณ์ คะแนน ความหมายสำหรับ user
กำไรลอยตัวดีขึ้นระหว่างถือไม้ + unrealized_delta x 0.1 ให้ hint เล็ก ๆ ว่าทิศทางเริ่มถูก แต่ไม่เยอะจน agent เล่นเกมกับ unrealized P/L
ปิดไม้แล้วได้ net P/L + net_pnl x 50 นี่คือ reward หลัก ถ้าปิดได้กำไรจะบวก ถ้าปิดขาดทุนจะลบตามเงินจริง
ปิดไม้กำไรสุทธิมากกว่า 0.5% + 0.01 ให้ bonus เฉพาะไม้ที่ชนะจริง ไม่ใช่กำไรเล็กมากที่อาจแพ้ต้นทุน
เปิดไม้ใหม่ - 0.005 กันการเทรดถี่เกินไป agent ต้องมั่นใจพอถึงจะคุ้มค่าปรับนี้
ไม่ถือ position และเลือก Hold - 0.0001 กัน agent เรียนรู้ว่าไม่ต้องทำอะไรเลยตลอดเวลา
ถือไม้นานเกิน 70% ของ max_hold - 0.0002 ต่อ step กระตุ้นให้ปิดไม้ตามเวลา ไม่ปล่อยลากยาวแบบ buy-and-hold
กำไรลอยตัวเคยเกิน 0.2% แล้วคืนกลับมากกว่าครึ่ง - 0.001 สอนให้ agent ไม่ปล่อยกำไรที่ดีไหลกลับมากเกินไป

ต้นทุนที่ env คิดอยู่

Spread

เข้าออกโดน spread

ค่า default คือ 0.0002 หรือประมาณ 0.02% โดย Buy เข้าแพงขึ้น และ Sell เข้า/ออกตามฝั่งราคา

Commission

ปิดไม้แล้วหัก commission

ค่า default คือ 0.0001 หรือประมาณ 0.01% ถูกนำไปหักใน net_pnl

ภาพรวมต่อการเทรดหนึ่งรอบ ต้นทุนคร่าว ๆ อยู่แถว 0.05% ดังนั้น reward จึงให้ bonus เฉพาะไม้ที่กำไรสุทธิเกิน 0.5% เพื่อไม่ให้ agent ดีใจกับกำไรเล็กที่อาจไม่ชนะต้นทุนจริง

Realized vs MTM

Recommended

realized

ให้ reward หลักตอนปิดไม้ เหมาะกับการสอนให้ agent สนใจผลลัพธ์จริงและลดพฤติกรรมถือยาวเพื่อกินคะแนนลอยตัว

Legacy

mtm

ให้คะแนนตาม mark-to-market ทุก step ใช้ได้ แต่เสี่ยงทำให้ agent ติดนิสัย buy-and-hold หรือไล่ unrealized P/L

คำอธิบายสำหรับสอน user

ให้จำง่าย ๆ ว่า RL agent ไม่ได้เรียนจากคำว่า UP หรือ DOWN แบบโมเดล supervised ML แต่เรียนจากประสบการณ์ว่า กด Buy, Sell, Hold, Close แล้วสุดท้าย equity ดีขึ้นหรือแย่ลง

Reward ปัจจุบันจึงออกแบบให้คล้ายการเทรดจริง: เปิดไม้มีต้นทุน, ถือยาวมีความเสี่ยง, ปล่อยกำไรคืนมีค่าปรับ และปิดไม้กำไรจริงเท่านั้นที่ได้คะแนนใหญ่

ถ้า train แล้ว reward สูงขึ้น แต่ backtest ยังขาดทุน ให้ยึด backtest และ walk-forward เป็นตัวตัดสินสุดท้าย เพราะ reward เป็นสัญญาณฝึก ไม่ใช่ผลกำไรจริงของระบบ
