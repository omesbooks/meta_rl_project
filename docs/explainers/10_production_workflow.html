🔄 3 Loops ทำงานพร้อมกัน

แต่ละ loop ทำงานอิสระ ไม่ block กัน
⚡

Trading Loop

ทุก H1 bar (รอ candle ปิด)
1mt5.copy_rates_from_pos() ดึง 250 bars ล่าสุด
2คำนวณ 35 features เหมือน DataCollector_v2
3Normalize ด้วย rl_v3_norm.csv
4สร้าง state (window 10 + position info)
5model.predict(state) → action + confidence
6Confidence filter: skip ถ้า conf < 0.85
7Risk check: position size, max DD
8mt5.order_send() ส่ง order
9Log → SQLite (state + action + reward)
10Sleep รอ bar ถัดไป
📊

Monitoring Loop

ทุก 5 นาที
1อ่าน live_trades.db
2คำนวณ rolling metrics:
WR, PF, MaxDD (last 50 trades)
3เปรียบเทียบกับ backtest baseline
4Drift detection:
ถ้า PF < 0.9 ติด 30 trades → 🚨 alert
5ส่ง notification (Telegram/Discord/Email)
6Auto-pause:
ถ้า DD > 15% → หยุดเทรด รอ manual review
7Update dashboard (Grafana/HTML)
8Trigger fine-tune ถ้าเงื่อนไขถึง
🔄

Fine-tune Loop

ทุก 1-3 เดือน หรือ on-trigger
1ดึง data ล่าสุด 3 เดือนจาก MT5
mt5.copy_rates_range()
2คำนวณ features + relabel
3rl_finetune.py ด้วย mode=mixed
30% old + 70% new, 50k steps
4Walk-forward validate model ใหม่
5Gate: deploy เฉพาะถ้า WF ผ่าน >= 4/5 windows
6Backup model เดิม → deploy model ใหม่
rl_v3 → rl_v3_archive_2026Q2
7Hot reload (Trading Loop โหลด model ใหม่)
8Log + alert: "Deployed rl_v4"

🛡️ Risk Management (3 ชั้น)

L1

🔴 Hard Stop (Critical)

account_equity < initial * 0.85 → ปิดทุก position + หยุดเทรด
ป้องกัน catastrophic loss · ต้อง manual restart หลัง review

L2

🟡 Soft Pause (Warning)

rolling_pf < 0.9 (last 30 trades) → หยุดเปิด trade ใหม่ 24 ชม
ปิด open positions ตามปกติ · trigger auto fine-tune

L3

🟢 Position Sizing (Always-on)

Per trade: risk <= account * 1% · Max open positions: 3
Lot size = (risk_amt) / (SL_distance × tick_value)

💻 Starter Code — live_trader.py

หมายเหตุ: นี่คือ skeleton — ของจริงต้องเพิ่ม error handling, logging, reconnect logic, etc.
# live_trader.py — Main trading loop import MetaTrader5 as mt5 import time, sqlite3, numpy as np, pandas as pd from stable_baselines3 import PPO from trading_env import TradingEnv from features import calc_features # เขียนแยก, reuse จาก v2 # === Config === SYMBOL = "XAUUSDm" TIMEFRAME = mt5.TIMEFRAME_H1 MODEL_PATH = "rl_v3.zip" NORM_PATH = "rl_v3_norm.csv" CONF_THRESHOLD = 0.85 MAX_DD_PCT = 0.15 RISK_PER_TRADE = 0.01 # 1% # === Init === mt5.initialize() model = PPO.load(MODEL_PATH) norm = pd.read_csv(NORM_PATH, index_col=0) db = sqlite3.connect("live_trades.db") # === Main Loop === while True: try: # 1. Wait for new bar wait_for_new_bar(SYMBOL, TIMEFRAME) # 2. Pull data rates = mt5.copy_rates_from_pos(SYMBOL, TIMEFRAME, 0, 250) df = pd.DataFrame(rates) # 3. Calc features df = calc_features(df) # 4. Normalize for c in norm.index: df[c] = (df[c] - norm.at[c, "mean"]) / norm.at[c, "std"] # 5. Build state (window=10 + position info) state = build_state(df, position=get_position(SYMBOL)) # 6. Predict + confidence action, conf = predict_with_confidence(model, state) # 7. Filter + Risk Check if conf < CONF_THRESHOLD: continue if current_dd() > MAX_DD_PCT: close_all_and_pause() continue # 8. Calculate position size sl, tp = calc_sl_tp(action, df) lots = calc_lot_size(sl, RISK_PER_TRADE) # 9. Send order if action == 1: send_buy(SYMBOL, lots, sl, tp) elif action == 2: send_sell(SYMBOL, lots, sl, tp) elif action == 3: close_all(SYMBOL) # 10. Log log_to_db(db, state, action, conf) except Exception as e: log_error(e) time.sleep(60)

✅ Deployment Checklist

📋 ก่อน go live
✓Model validated ผ่าน walk-forward อย่างน้อย 4/5 windows PF > 1.0
✓Demo account testing อย่างน้อย 1 เดือน · trades ≥ 30 ครั้ง
✓Demo vs backtest เทียบ — ผลใกล้กัน (slippage acceptable)
✓Risk parameters set: RISK_PER_TRADE, MAX_DD, MAX_POSITIONS
✓Reconnect logic ถ้า MT5 disconnect → retry exponential backoff
✓Notification setup: Telegram bot / Discord webhook
✓Database backup ทุกวัน
✓Watchdog — ถ้า Python หยุด → restart อัตโนมัติ (systemd/Task Scheduler)
✓Live trading start ด้วย lot 0.01 ก่อน · scale up ค่อยๆ
✓Fine-tune schedule อย่างน้อยทุก 3 เดือน
📋 ต่อเนื่อง (after deploy)
✓Daily: เช็ค dashboard · ดูว่า trade เกิดถูกต้อง
✓Weekly: ตรวจ rolling PF/WR · วิเคราะห์ trades แย่ๆ
✓Monthly: เปรียบ live vs backtest baseline · ตัดสินใจ fine-tune
✓Quarterly: Fine-tune ใหม่ + walk-forward + deploy ถ้าผ่าน
✓Yearly: Retrain scratch เพื่อ reset drift accumulation

⚠️ Common Pitfalls ใน Production

🚨 ระวัง 1: Model เก่าไม่อัพเดต norm stats
ถ้าเปลี่ยน norm หรือ retrain → ต้องอัพเดตทั้ง .zip และ _norm.csv พร้อมกัน
🚨 ระวัง 2: Slippage / Spread ในตลาดจริง
Backtest อาจไม่นับ → live แย่กว่าที่คาด · เพิ่ม buffer ใน confidence threshold
🚨 ระวัง 3: Bar timing ระหว่าง broker
Broker A bar ปิดเวลา 12:00:00 · Broker B 12:00:05 · ต้อง sync ให้ดี
🚨 ระวัง 4: Connection ขาด
Python disconnect แต่ MT5 ยัง run → mismatch · Watchdog EA ช่วย detect
🚨 ระวัง 5: Fine-tune จู่ๆ deploy โดยไม่ validate
ห้าม auto-deploy โดยไม่ผ่าน walk-forward gate · เสี่ยง deploy model พังลงตลาดจริง

🔄 Fine-tune Trigger Logic

ตัดสินใจเมื่อไหร่ควร fine-tune
🎯 เงื่อนไข trigger fine-tune
⏰Schedule: ทุก 3 เดือนอัตโนมัติ
📉Performance drop: live PF < 0.9 ติด 50 trades
📊Market regime change: ATR mean > 2× backtest baseline
🔄Manual: เห็น drift ในข่าวเศรษฐกิจสำคัญ (FOMC, NFP)
✅ Gate ก่อน deploy fine-tuned model
1Walk-forward 5 windows ผ่าน ≥ 4/5
2PF บน latest data > 1.05
3Action distribution ใกล้ original (อย่ามี mode shift แปลกๆ)
4Demo testing 1 สัปดาห์ ก่อน switch live
