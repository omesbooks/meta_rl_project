Quick Start สำหรับ User ใหม่

Goal

เริ่มจากข้อมูลที่ถูกต้อง

ให้ user เริ่มจาก DataCollector_RL ใน MT5 แล้ว import CSV เข้าโปรเจกต์ผ่านหน้า Data Tools

Train

Train แล้วดูผล ไม่ใช่ train แล้วเชื่อทันที

โมเดล RL ต้องผ่าน quick test, backtest, และ walk-forward ก่อนคิดเรื่อง export

Deploy

Export เฉพาะโมเดลที่ผ่าน gate

ถ้า return/PF/drawdown ยังแย่ ให้กลับไปปรับข้อมูลหรือ train ใหม่ ไม่ควร export ไป MT5

จำง่าย ๆ: Data -> Quality/Regime Check -> Train -> Backtest -> Walk-Forward -> Export. ถ้าขั้นไหนไม่ผ่าน ให้ย้อนกลับไปแก้ข้อมูลหรือ hyperparameters ก่อน ไม่ข้ามไป deploy

ศัพท์พื้นฐานที่ควรรู้ก่อนเริ่มใช้ Dashboard

คำเหล่านี้จะเจอบ่อยในหน้า Train, Backtest, Walk-Forward และเอกสารเจาะลึก ถ้า user เป็น beginner ให้ปูความหมายตรงนี้ก่อน เพื่อให้เข้าใจว่าแต่ละขั้นกำลังพิสูจน์อะไร

Edge

มี edge

แปลว่าโมเดลหรือระบบมีความได้เปรียบทางสถิติ ไม่ใช่แค่ชนะบางไม้เพราะโชค ตัวอย่างเช่น ทดสอบหลายช่วงแล้วยังมี PF ดี, drawdown รับได้ และไม่ได้กำไรจาก trade เดียว

Validation

Backtest

การเอาโมเดลที่ train แล้วไปจำลองเทรดกับข้อมูลย้อนหลัง เพื่อดู return, profit factor, drawdown, win rate และจำนวน trade ก่อนใช้งานจริง

Robustness

Walk-Forward

การแบ่งเวลาเป็นหลาย window แล้ว train/test ทีละช่วง เพื่อเช็คว่าโมเดลหรือ recipe ไม่ได้ดีเฉพาะช่วงเดียวของตลาด

Market

Regime Check

การตรวจว่าตลาดมีช่วงนิสัยต่างกันหรือมีจุดเปลี่ยนใหญ่ไหม เช่น ก่อน/หลัง crisis หรือ event สำคัญ ถ้าเจอ breakpoint อาจใช้เป็นจุดตัดข้อมูลก่อน train

Feature

Correlation

ค่าที่บอกว่า feature สองตัวเคลื่อนไหวคล้ายกันแค่ไหน ถ้าสูงมาก เช่น 0.85 ขึ้นไป อาจแปลว่าข้อมูลซ้ำกันเยอะและทำให้โมเดลเรียนรู้หนักเกินจำเป็น

Feature

Redundant

feature ที่ให้ข้อมูลซ้ำกับ feature อื่นมากเกินไป การ Clean Redundant Features คือการลด input ซ้ำ ๆ เพื่อลด noise และลดโอกาส overfit

Training

Window Size

จำนวนแท่งเทียนย้อนหลังที่โมเดลเห็นในหนึ่งครั้ง เช่น 10 หมายถึงให้โมเดลดูบริบท 10 bars ล่าสุดก่อนตัดสินใจ Hold, Buy, Sell หรือ Close

Training

Training Steps

จำนวนครั้งที่ agent ได้ฝึกตัดสินใจใน environment steps มากขึ้นไม่เท่ากับดีขึ้นเสมอ ต้องดู log, backtest และ walk-forward ประกอบ

Filter

Confidence Threshold

ค่า gate ใน Backtest/Live ที่ใช้กรองเฉพาะ action Buy/Sell เช่น 0.85 ถ้าต่ำกว่านี้ระบบจะ skip signal แต่ action Close ไม่ควรถูกกรอง ตอน Train/Quick eval ของ PPO ไม่ได้ใช้ filter นี้ จึงควรเทียบผลแบบ conf = 0 ก่อน แล้วค่อย sweep threshold หลัง train

Improve

Fine-tune

การเอาโมเดลเดิมมาฝึกต่อด้วยข้อมูลใหม่ เหมาะเมื่อโมเดลเดิมยังพอมี edge แต่ตลาดเปลี่ยนหรือมีข้อมูลล่าสุดเพิ่มเข้ามา

Gate

โมเดลผ่าน gate

หมายถึงผ่านเงื่อนไขตรวจสอบในแต่ละด่าน เช่น data ใช้ได้, train healthy, backtest รับได้, walk-forward ไม่พังหลาย window ยังไม่ใช่การรับประกันว่าจะกำไรจริง

Library

Models

หน้าเก็บและตรวจรายการโมเดลที่ train แล้ว รวมถึง artifact ที่เกี่ยวข้อง เช่น model zip, normalization stats, params และผล backtest

Flow

Pipeline

หน้าที่รันหลายขั้นต่อกันแบบรวดเดียว เช่น dataset -> train -> backtest -> chart/report เหมาะกับ flow ที่ตั้งค่าพร้อมแล้ว ส่วนหน้า Train/Backtest แยกเหมาะกับการคุมรายละเอียดเอง

ประโยคสอน user แบบสั้น: เรายังไม่เชื่อโมเดลจากการ train สำเร็จ แต่จะเชื่อมากขึ้นเมื่อผ่าน gate ด้วยหลักฐานจาก Backtest และ Walk-Forward

Main Flow การใช้งาน Dashboard

1
Data Prep

เตรียมข้อมูลจาก MT5

รัน DataCollector_RL ใน MT5 เพื่อสร้าง CSV แล้วเปิดหน้า Data Tools เพื่อ import ไฟล์เข้ามาในโปรเจกต์

  • เลือก CSV จาก Common\Files
  • ตั้ง output basename เช่น training_data_gu_h4_dataset.csv
  • กด Import CSV + params
  • ถ้ามี .params.json ให้เก็บคู่กับ CSV เพื่อ parity ตอน export/deploy
Deep dive ต่อ 02_data_prep_import_detail.html อธิบาย DataCollector_RL, Common Files, .params.json, และการเช็คว่า CSV มีข้อมูลจริง
2
Quality Check

เช็คคุณภาพข้อมูลและ Feature

ก่อน train รอบจริง ให้เช็คทั้งคุณภาพไฟล์, feature ที่ซ้ำกัน และ regime ของตลาด เพื่อให้ dataset ที่เอาไปฝึกไม่ปนช่วงตลาดที่คนละนิสัยกันมากเกินไป

  • ใช้ Show Correlation Matrix เพื่อดู feature ที่สัมพันธ์กันสูง
  • ใช้ Clean Redundant Features เมื่ออยากลด input ที่ซ้ำกัน
  • ใช้ Regime Check ก่อน train ถ้า dataset ย้อนหลังยาว หรือสงสัยว่าตลาดมี structural break
  • ถ้า Regime Check เจอจุดเปลี่ยนสำคัญ ให้สร้าง cutoff dataset ก่อนนำไป train
  • Split Train/Test by Date ใช้เมื่ออยากแยกไฟล์ train/test ชัดเจน
  • สำหรับ RL basic flow สามารถใช้ Train % ในหน้า Train/Pipeline แทน split เป็นไฟล์ได้
Deep dive ต่อ 03_quality_check_detail.html อธิบาย Data Tools, correlation threshold, Regime Check, breakpoint table, train cutoff, และ train/test discipline
3
Training

Train โมเดล RL

เลือกหน้า Train เมื่อต้องการควบคุมการ train เอง หรือหน้า Pipeline ถ้าต้องการรัน flow ยาวแบบรวดเดียว

  • เลือก dataset CSV
  • เลือก PPO เป็นค่าแนะนำ
  • ใช้ reward mode realized
  • ตั้ง Train %, Window Size, Max Hold, Training Steps
  • ดู reward trend และ PPO health metrics ระหว่าง train
Deep dive ต่อ 04_train_page_detail.html อธิบาย PPO settings, reward, train pct, window size, max hold, และการอ่าน train log
4
Validation

Backtest โมเดล

ใช้หน้า Backtest เพื่อดูว่าโมเดลที่ train แล้วทำงานกับข้อมูลทดสอบได้จริงหรือไม่

  • เลือก model และ CSV สำหรับ backtest
  • เริ่มด้วย Confidence = 0 เพื่อเทียบ baseline กับ quick eval ตอน train
  • ถ้าจะใช้ live gate ให้ sweep confidence threshold เช่น 0.3, 0.5, 0.85 แทนการ fix ค่าเดียวทุกโมเดล
  • ดู return, profit factor, win rate, max drawdown, จำนวน trades
  • ถ้าผลลบหนักหรือ PF ต่ำกว่า 1 ให้กลับไปแก้ data/train
Deep dive ต่อ 05_backtest_page_detail.html อธิบาย confidence filter, การเทียบกับ train baseline, backtest modes, metric interpretation, และ common failure patterns
5
Robustness

Walk-Forward

ถ้า backtest ดูดี ให้ใช้ Walk-Forward เพื่อเช็คว่า edge ไม่ได้ดีแค่ช่วงเดียว และโมเดลยังพอรับมือกับหลายช่วงเวลาได้

  • Walk-Forward เทรน/ทดสอบหลายช่วงเวลา
  • ถ้า window ส่วนใหญ่ไม่ผ่าน ให้ถือว่าโมเดลยังไม่ robust
  • ถ้าผลล้มหลาย window ให้ย้อนกลับไปดู data quality, Regime Check, หรือ fine-tune ใหม่
Deep dive ต่อ 06_walk_forward_detail.html อธิบาย walk-forward custom logic, window split, rolling vs anchored, และ robust verdict
6
Improve

Fine-tune หรือ Train ใหม่

ถ้าโมเดลเดิมยังพอมี edge แต่ตลาดเปลี่ยน ให้ fine-tune ด้วยข้อมูลใหม่ ถ้าโมเดลแย่มากให้ train ใหม่จาก dataset ที่สะอาดกว่า

  • Fine-tune เหมาะกับการอัปเดตจากโมเดลเดิม
  • Train ใหม่เหมาะเมื่อเปลี่ยน symbol/timeframe/features หรือโมเดลเดิมไม่ดี
  • หลัง fine-tune ต้อง backtest และ walk-forward ใหม่
Deep dive ต่อ อธิบาย train vs fine-tune, catastrophic forgetting, และ quarterly update flow
7
Export

Export ไป MT5

เมื่อโมเดลผ่าน gate แล้ว ใช้หน้า Data Tools หรือ Export flow เพื่อแปลง PPO เป็น ONNX และสร้างไฟล์ MT5

  • เลือก source model
  • ตั้ง deploy name
  • export เป็น .onnx, _config.mqh, และ _EA.mq5
  • นำไฟล์ไปทดสอบใน MT5 Strategy Tester ก่อนใช้งานจริง
Deep dive ต่อ อธิบาย ONNX export, MT5 folder structure, config parity, และ strategy tester checklist
8
Manage

จัดการโมเดลและตั้งค่า

ใช้หน้า Models เพื่อดู model library และใช้ Settings สำหรับ branding/API keys/ค่าระบบ

  • ดูรายชื่อ model ที่ train แล้ว
  • ตรวจ artifacts ที่เกี่ยวกับ model
  • ตั้งค่า branding และ API key สำหรับ Gemini ใน Regime Check
Deep dive ต่อ อธิบาย model artifact set, cleanup policy, settings page, และ API key handling

หน้า ใช้เมื่อ ผลลัพธ์ที่ควรได้

Decision Gates: เมื่อไหร่ควรไปต่อ

Gate ผ่าน ยังไม่ผ่าน
Data ready Go CSV มี rows, timestamp เรียงได้, feature ครบ, มี params ถ้าจะ deploy Fix กลับไป DataCollector_RL หรือ Data Tools import ใหม่
Pre-train regime checked Go dataset เหมาะกับช่วงตลาดที่จะ train หรือสร้าง cutoff dataset แล้ว Review ใช้ Regime Check ก่อน train ถ้าข้อมูลยาว/เก่า/มี event ใหญ่
Train healthy Go reward ไม่ collapse, PPO metrics ไม่ผิดปกติ, ไม่มี error/nan Tune ลด learning rate, steps, feature, หรือเปลี่ยน dataset
Backtest acceptable Go PF > 1, drawdown รับได้, trades ไม่บางเกินไป Stop ห้าม export ใช้งานจริง ให้กลับไป data/train
Walk-forward robust Go หลาย window ผ่าน ไม่ใช่ดีแค่ช่วงเดียว Review เช็ค regime shift หรือ fine-tune ใหม่
MT5 test ready Go export สำเร็จ EA compile/tester ได้ Fix เช็ค ONNX, config, feature parity, MT5 logs

ข้อควรระวังเวลาสอน User

Do not skip

อย่าข้าม validation

โมเดลที่ train สำเร็จไม่ได้แปลว่าเทรดดี ต้องดู backtest/walk-forward เสมอ

Do not mix

อย่าสับสน RL กับ PyCaret

RL ไม่ได้ใช้ UP/DOWN/FLAT target เป็นหัวใจหลัก แต่เรียนจาก action และ reward ใน environment

ถ้า user ถามว่า “พร้อม deploy หรือยัง” ให้ตอบจาก evidence เท่านั้น: training log, backtest result, walk-forward result และ MT5 tester result ไม่ใช่จากการที่ไฟล์ export สำเร็จเฉย ๆ
