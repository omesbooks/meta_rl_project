Quick Start สำหรับ User ใหม่

Goal

เริ่มจากข้อมูลที่ถูกต้อง

ให้ user เริ่มจาก DataCollector_RL ใน MT5 แล้ว import CSV เข้าโปรเจกต์ผ่านหน้า Data Tools

Train

Train แล้วดูผล ไม่ใช่ train แล้วเชื่อทันที

โมเดล RL ต้องผ่าน quick test, backtest, และ walk-forward ก่อนคิดเรื่อง export

Deploy

Export เฉพาะโมเดลที่ผ่าน gate

ถ้า return/PF/drawdown ยังแย่ ให้กลับไปปรับข้อมูลหรือ train ใหม่ ไม่ควร export ไป MT5

จำง่าย ๆ: Data -> Train -> Backtest -> Walk-Forward -> Export. ถ้าขั้นไหนไม่ผ่าน ให้ย้อนกลับไปแก้ข้อมูลหรือ hyperparameters ก่อน ไม่ข้ามไป deploy

Main Flow การใช้งาน Dashboard

1
Data Prep

เตรียมข้อมูลจาก MT5

รัน DataCollector_RL ใน MT5 เพื่อสร้าง CSV แล้วเปิดหน้า Data Tools เพื่อ import ไฟล์เข้ามาในโปรเจกต์

  • เลือก CSV จาก Common\Files
  • ตั้ง output basename เช่น training_data_gu_h4_dataset.csv
  • กด Import CSV + params
  • ถ้ามี .params.json ให้เก็บคู่กับ CSV เพื่อ parity ตอน export/deploy
Deep dive ต่อ 15_data_prep_import_detail.html อธิบาย DataCollector_RL, Common Files, .params.json, และการเช็คว่า CSV มีข้อมูลจริง
2
Quality Check

เช็คคุณภาพข้อมูลและ Feature

ถ้า dataset มี feature จำนวนมาก ให้ใช้ Feature Analysis & Cleanup เพื่อดู correlation และลด feature ซ้ำ

  • ใช้ Show Correlation Matrix เพื่อดู feature ที่สัมพันธ์กันสูง
  • ใช้ Clean Redundant Features เมื่ออยากลด input ที่ซ้ำกัน
  • Split Train/Test by Date ใช้เมื่ออยากแยกไฟล์ train/test ชัดเจน
  • สำหรับ RL basic flow สามารถใช้ Train % ในหน้า Train/Pipeline แทน split เป็นไฟล์ได้
Deep dive ต่อ อธิบาย Data Tools แต่ละโมดูล, correlation threshold, และ train/test discipline
3
Training

Train โมเดล RL

เลือกหน้า Train เมื่อต้องการควบคุมการ train เอง หรือหน้า Pipeline ถ้าต้องการรัน flow ยาวแบบรวดเดียว

  • เลือก dataset CSV
  • เลือก PPO เป็นค่าแนะนำ
  • ใช้ reward mode realized
  • ตั้ง Train %, Window Size, Max Hold, Training Steps
  • ดู reward trend และ PPO health metrics ระหว่าง train
Deep dive ต่อ อธิบาย PPO settings, reward, train pct, window size, max hold, และการอ่าน train log
4
Validation

Backtest โมเดล

ใช้หน้า Backtest เพื่อดูว่าโมเดลที่ train แล้วทำงานกับข้อมูลทดสอบได้จริงหรือไม่

  • เลือก model และ CSV สำหรับ backtest
  • ตั้ง confidence threshold เช่น 0.85
  • ดู return, profit factor, win rate, max drawdown, จำนวน trades
  • ถ้าผลลบหนักหรือ PF ต่ำกว่า 1 ให้กลับไปแก้ data/train
Deep dive ต่อ อธิบาย confidence filter, backtest modes, metric interpretation, และ common failure patterns
5
Robustness

Walk-Forward และ Regime Check

ถ้า backtest ดูดี ให้ใช้ Walk-Forward เพื่อเช็คว่า edge ไม่ได้ดีแค่ช่วงเดียว และใช้ Regime Check เมื่อตลาดเปลี่ยน regime

  • Walk-Forward เทรน/ทดสอบหลายช่วงเวลา
  • Regime Check ช่วยหาจุดเปลี่ยนโครงสร้างตลาด
  • ถ้า window ส่วนใหญ่ไม่ผ่าน ให้ถือว่าโมเดลยังไม่ robust
  • ถ้าเจอ regime shift ให้ตัด dataset หรือ fine-tune ใหม่
Deep dive ต่อ อธิบาย walk-forward custom logic, regime methods, breakpoint table, และ train cutoff
6
Improve

Fine-tune หรือ Train ใหม่

ถ้าโมเดลเดิมยังพอมี edge แต่ตลาดเปลี่ยน ให้ fine-tune ด้วยข้อมูลใหม่ ถ้าโมเดลแย่มากให้ train ใหม่จาก dataset ที่สะอาดกว่า

  • Fine-tune เหมาะกับการอัปเดตจากโมเดลเดิม
  • Train ใหม่เหมาะเมื่อเปลี่ยน symbol/timeframe/features หรือโมเดลเดิมไม่ดี
  • หลัง fine-tune ต้อง backtest และ walk-forward ใหม่
Deep dive ต่อ อธิบาย train vs fine-tune, catastrophic forgetting, และ quarterly update flow
7
Export

Export ไป MT5

เมื่อโมเดลผ่าน gate แล้ว ใช้หน้า Data Tools หรือ Export flow เพื่อแปลง PPO เป็น ONNX และสร้างไฟล์ MT5

  • เลือก source model
  • ตั้ง deploy name
  • export เป็น .onnx, _config.mqh, และ _EA.mq5
  • นำไฟล์ไปทดสอบใน MT5 Strategy Tester ก่อนใช้งานจริง
Deep dive ต่อ อธิบาย ONNX export, MT5 folder structure, config parity, และ strategy tester checklist
8
Manage

จัดการโมเดลและตั้งค่า

ใช้หน้า Models เพื่อดู model library และใช้ Settings สำหรับ branding/API keys/ค่าระบบ

  • ดูรายชื่อ model ที่ train แล้ว
  • ตรวจ artifacts ที่เกี่ยวกับ model
  • ตั้งค่า branding และ API key สำหรับ Gemini ใน Regime Check
Deep dive ต่อ อธิบาย model artifact set, cleanup policy, settings page, และ API key handling

Page Map: แต่ละหน้าใน Dashboard ใช้ทำอะไร

หน้า ใช้เมื่อ ผลลัพธ์ที่ควรได้
Data Tools เตรียม CSV, split, clean feature, export ONNX Dataset ที่พร้อม train หรือไฟล์ MT5 สำหรับ deploy
Pipeline ต้องการรัน train -> backtest -> chart/report แบบ flow เดียว Model, backtest result, chart/report และ log รวม
Train ต้องการ train แบบควบคุมละเอียด .zip model, norm stats, best model folder, train log
Backtest ต้องการทดสอบโมเดลกับ dataset Return, PF, DD, trades, equity curve
Walk-Forward ต้องการพิสูจน์ robustness หลายช่วงเวลา ผลลัพธ์ราย window และ verdict ว่า robust หรือไม่
Fine-tune ต้องการอัปเดตโมเดลเดิมด้วยข้อมูลใหม่ โมเดลใหม่ที่ต่อยอดจาก base model
Analyze ต้องการวิเคราะห์ confidence/accuracy ความเข้าใจว่า threshold ไหนน่าใช้
Regime Check สงสัยว่าตลาดเปลี่ยน regime หรือ dataset เก่าเกินไป breakpoints, event labels, และ cutoff dataset สำหรับ train ใหม่
Models ต้องการดู/จัดการ model library ภาพรวม model และ artifacts ที่มีอยู่
Settings ตั้งค่า app, branding, API keys ค่าระบบพร้อมใช้งาน

Decision Gates: เมื่อไหร่ควรไปต่อ

Gate ผ่าน ยังไม่ผ่าน
Data ready Go CSV มี rows, timestamp เรียงได้, feature ครบ, มี params ถ้าจะ deploy Fix กลับไป DataCollector_RL หรือ Data Tools import ใหม่
Train healthy Go reward ไม่ collapse, PPO metrics ไม่ผิดปกติ, ไม่มี error/nan Tune ลด learning rate, steps, feature, หรือเปลี่ยน dataset
Backtest acceptable Go PF > 1, drawdown รับได้, trades ไม่บางเกินไป Stop ห้าม export ใช้งานจริง ให้กลับไป data/train
Walk-forward robust Go หลาย window ผ่าน ไม่ใช่ดีแค่ช่วงเดียว Review เช็ค regime shift หรือ fine-tune ใหม่
MT5 test ready Go export สำเร็จ EA compile/tester ได้ Fix เช็ค ONNX, config, feature parity, MT5 logs

ข้อควรระวังเวลาสอน User

Do not skip

อย่าข้าม validation

โมเดลที่ train สำเร็จไม่ได้แปลว่าเทรดดี ต้องดู backtest/walk-forward เสมอ

Do not mix

อย่าสับสน RL กับ PyCaret

RL ไม่ได้ใช้ UP/DOWN/FLAT target เป็นหัวใจหลัก แต่เรียนจาก action และ reward ใน environment

ถ้า user ถามว่า “พร้อม deploy หรือยัง” ให้ตอบจาก evidence เท่านั้น: training log, backtest result, walk-forward result และ MT5 tester result ไม่ใช่จากการที่ไฟล์ export สำเร็จเฉย ๆ
