Flow Step 3 · Training

Train โมเดล RL อย่างมีวินัย

บทนี้อธิบายหน้า Train: เลือก dataset, ตั้งค่าหลักของ PPO, อ่าน log ระหว่าง train และรู้ว่าไฟล์ output แต่ละตัวเอาไปใช้ทำอะไรต่อ

เป้าหมาย ขั้นตอนใช้งาน ไฟล์ที่ต้องใช้ ค่าหลัก Advanced PPO อ่าน Log ส่งต่อ Validation

Training คืออะไรใน flow นี้

Input

ใช้ dataset ที่ผ่าน Quality Check

ควรใช้ CSV ที่ timestamp ใช้ได้, feature ไม่เสีย, correlation ไม่หนักเกิน, และถ้าจะ deploy ควรมี .params.json คู่กัน

Learn

ให้ agent ฝึกจาก state → action → reward

agent เห็น feature window และสถานะ position แล้วเลือก action: Hold, Buy, Sell หรือ Close

Not Done

Train สำเร็จยังไม่แปลว่าใช้ได้

หลัง train ต้องผ่าน Backtest และ Walk-Forward ก่อนคิดเรื่อง export/deploy

ขั้นตอน Train ใน Dashboard

เวลาใช้สอน user ให้เริ่มจาก flow ตรงนี้ก่อน เพื่อให้เห็นว่าหน้า Train ต้องเดินจาก dataset ที่ผ่าน Quality Check ไปเป็น model artifact อย่างไร หลังจากนั้นค่อยลงรายละเอียดว่าไฟล์ output, Train %, window size, max hold และ PPO settings แต่ละตัวคืออะไร

1
Dataset

เลือก CSV ที่ผ่าน Step 2 แล้ว

เลือกไฟล์จากหน้า Train หรือใช้ไฟล์ที่ Regime Check ส่งมาให้ เช่น training_data_usdjpy_h4_dataset_from_2020-03-01.csv

2
Params

เช็ค .params.json

ถ้า dataset ไม่มี sidecar ให้กด Attach .params เฉพาะเมื่อมั่นใจว่าเป็น params ของ dataset ชุดเดียวกัน

3
Settings

ตั้งค่าหลักก่อน

ใช้ PPO, reward realized, train pct 85, window 10, max hold 30, และตั้งชื่อโมเดลให้ชัด

4
Run

กด Start Training

ระบบจะรัน rl_train.py, split แบบ time-based, normalize จาก train stats เท่านั้น, train, save model และ quick test บน eval/test

5
Review

อ่านผลก่อน Validation

Train log ที่ไม่มี error ไม่พอ ต้องดู reward trend, PPO health metrics, quick test และ artifact ที่สร้างครบหรือไม่

ไฟล์ที่ Train ใช้และสร้างออกมา

ในหน้า Train ค่า Model Name จะกลายเป็นชื่อโฟลเดอร์และชื่อไฟล์ของโมเดล เช่นตั้งชื่อ rl_usdjpy_h4_v1 แล้วกด Start Training สำเร็จ ระบบจะสร้างไฟล์หลักไว้ใต้ project root ที่ artifacts/models/rl_usdjpy_h4_v1/

กลุ่มไฟล์ Path / ชื่อไฟล์ที่ได้ ใช้ทำอะไร
Dataset input training_data_usdjpy_h4_dataset.csv ใน project root เป็นข้อมูลหลักที่ rl_train.py ใช้ train/eval แบบ time-based split
Dataset sidecar training_data_usdjpy_h4_dataset.params.json ใน project root ถูก forward เป็น <model>.params.json เพื่อใช้ตอน export/deploy
Final model artifacts/models/<model>/<model>.zip ไฟล์โมเดล PPO หลัก ใช้เลือกในหน้า Backtest, Fine-tune และ Export
Normalization stats artifacts/models/<model>/<model>_norm.csv ช่วยให้ inference/backtest/export ใช้ scale เดียวกับตอน train
Params parity artifacts/models/<model>/<model>.params.json copy จาก dataset sidecar เพื่อให้ export/deploy ใช้ indicator setting ตรงกับ DataCollector_RL
Train metadata artifacts/models/<model>/<model>.train.json บันทึก setting ตอน train, path ของ artifact และข้อมูลช่วยตรวจย้อนหลังเมื่อผลลัพธ์ผิดปกติ
Best checkpoint artifacts/models/<model>/best/best_model.zip checkpoint ที่ EvalCallback มองว่าดีที่สุดระหว่าง train ใช้เทียบกับ final model ได้
Training logs artifacts/models/<model>/logs/ เก็บ log/eval output จาก Stable-Baselines3 สำหรับดูประวัติ training
อธิบายแบบง่าย: CSV คือบทเรียน, .params.json คือคู่มือบทเรียน, .zip คือสมองที่ train แล้ว, และ _norm.csv คือสูตรปรับสเกล input ให้สมองอ่านข้อมูลแบบเดิมทุกครั้ง
จำ path แบบสั้น: งาน train ใหม่จะอยู่ใน artifacts/models/<model>/ ส่วนไฟล์โมเดลเก่าที่เคยอยู่หน้า root เช่น <model>.zip, <model>_norm.csv หรือ <model>_best/best_model.zip ระบบยังพยายามค้นหาให้เพื่อรองรับงานเก่า แต่เวลาสอน user ให้ยึดโครงสร้างใหม่เป็นหลัก

ค่าหลักในหน้า Train

ค่า ความหมาย คำแนะนำเริ่มต้น
Algorithm ชนิด RL algorithm ที่ใช้ train ใช้ PPO (recommended) ก่อน เพราะเป็นค่าหลักของระบบนี้
Reward Mode วิธีให้คะแนน agent ใช้ realized เพราะเน้น reward จากการปิด trade และ net P/L จริง
Training Steps จำนวน step ที่ agent ได้ฝึก เริ่ม 200000; ชุดข้อมูลใหญ่ค่อยเพิ่มตามผล log/backtest
Window Size จำนวน bars ย้อนหลังที่ agent เห็นใน state เริ่ม 10; window ใหญ่ขึ้นทำให้ observation ใหญ่ขึ้นและ train หนักขึ้น
Max Hold จำนวน bars สูงสุดที่ถือ position ก่อน force close เริ่ม 30; ควรสัมพันธ์กับ timeframe และ style ของระบบ
Train % แบ่งข้อมูลแบบ time-based: ต้นไฟล์เป็น train, ท้ายไฟล์เป็น eval/test ใช้ 85 ได้สำหรับเริ่มต้น และห้ามคิดว่าเป็น random split
Model Name ชื่อ artifact ที่จะถูกสร้าง เช่น rl_usdjpy_h4_v1 ใส่ symbol/timeframe/version ให้ชัด และระวังชื่อซ้ำเพราะจะ overwrite

Advanced PPO Settings ควรแตะเมื่อไหร่

สำหรับ user ใหม่ ให้ใช้ค่า default ก่อน แล้วค่อยกลับมาแก้เมื่ออ่าน log/backtest แล้วมีเหตุผล

ค่า ใช้แก้อาการอะไร แนวทาง
learning_rate policy แกว่ง, KL/clip สูง, reward พังเร็ว ลดจาก 3e-4 เป็น 1e-4
clip_range ควบคุมไม่ให้ PPO เปลี่ยน policy แรงเกินในแต่ละ update เริ่ม 0.2; ไม่ควรปรับถ้ายังไม่อ่าน PPO metrics เป็น
ent_coef agent ไม่ explore หรือ action ซ้ำเกินไป เพิ่มเล็กน้อยเมื่อ exploration หาย แต่ต้อง backtest ยืนยัน
n_steps, batch_size, n_epochs ควบคุม rollout และ optimization batch_size ต้องไม่เกิน n_steps
gamma, gae_lambda, vf_coef discount, advantage smoothing และ critic/value loss ใช้ default ก่อน เว้นแต่มีเหตุผลจาก metric หรือ experiment
สอน user แบบง่าย: Advanced PPO ไม่ใช่ที่สำหรับเดาสุ่ม ถ้า train ไม่ดีให้กลับไปดู data, regime, feature และ reward trend ก่อน แล้วค่อยปรับ hyperparameters

อ่าน Log ระหว่าง Train

สิ่งที่เห็น แปลว่าอะไร ควรทำอะไร
ep_rew_mean ค่อย ๆ ดีขึ้น agent เริ่มได้ reward จาก environment Go ปล่อยจนจบ แล้วไป Backtest
reward collapse หรือแกว่งแรง อาจ learning rate สูง, data noisy, หรือ reward ถูก exploit Review ลด LR หรือกลับไปดู data/feature
approx_kl หรือ clip_fraction สูงผิดปกติ policy เปลี่ยนแรงเกินไป Tune ลด learning rate หรือใช้ Stable preset
มี NaN/error หรือ train data too small ข้อมูลหรือ setting ไม่พร้อม Stop กลับไป Quality Check

ส่งต่อไป Validation

เมื่อ train จบและ artifact สร้างครบ ให้ไปหน้า Backtest เพื่อพิสูจน์ด้วยผลลัพธ์เทรดจริงใน simulation

ก่อนออกจากหน้า Train ควรมี ถ้าไม่มี
Model artifact artifacts/models/<model>/<model>.zip train ยังไม่จบหรือ error
Normalization stats artifacts/models/<model>/<model>_norm.csv อย่า backtest/export เพราะ input scale อาจไม่ตรง
Params parity artifacts/models/<model>/<model>.params.json ถ้ามี sidecar ต้นทาง ยัง backtest ได้ แต่ก่อน deploy ควรแก้ parity ให้ครบ
ประโยคสอน user: การ train คือการสร้างผู้สมัครโมเดล ยังไม่ใช่การอนุมัติให้ใช้งานจริง ผู้สมัครต้องผ่าน Backtest และ Walk-Forward ก่อน
