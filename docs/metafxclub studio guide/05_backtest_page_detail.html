Flow Step 4 · Validation

Backtest โมเดลก่อนเชื่อผล Train

บทนี้อธิบายหน้า Backtest: เลือก model/dataset, ตั้ง confidence และ risk, อ่านผล return/PF/drawdown/trades และตัดสินใจว่าโมเดลควรไป Walk-Forward หรือย้อนกลับไปแก้ข้อมูล/train

เป้าหมาย ขั้นตอนใช้งาน ไฟล์ที่ใช้ Backtest Mode ค่าหลัก อ่าน Metrics ตัดสินใจ

Validation คืออะไรใน flow นี้

Proof

พิสูจน์ด้วย trade simulation

Backtest ใช้โมเดลที่ train แล้วมาตัดสินใจบนข้อมูลย้อนหลัง แล้วจำลอง position, confidence filter, risk และ exit logic

Evidence

ดูผลจาก metrics ไม่ใช่ความรู้สึก

ต้องดู return, profit factor, drawdown, จำนวน trades, win rate, exit reasons และ skipped signals

Gate

ยังไม่ใช่ deploy approval

Backtest ผ่านคือไป Walk-Forward ต่อได้ ไม่ใช่แปลว่าพร้อมเทรดจริงทันที

ขั้นตอน Backtest ใน Dashboard

เวลาใช้สอน user ให้เริ่มจาก flow ตรงนี้ก่อน เพื่อให้เห็นว่า Backtest คือการเอาโมเดลที่ train แล้วไปสอบกับข้อมูลย้อนหลัง ไม่ใช่การ train ซ้ำ หลังจากนั้นค่อยขยายความว่าไฟล์ input, mode, confidence, risk และ metrics แต่ละตัวคืออะไร

1
Model

เลือก model และ test dataset

เลือก <model>.zip ที่เพิ่ง train และเลือก dataset ที่ต้องการ test. ถ้าใช้ไฟล์เดียวกับ train ให้ใช้ Skip % เพื่อ test เฉพาะช่วงท้าย

2
Mode

เริ่มจาก Pure Agent

รัน Pure Agent เพื่อดูว่า policy เองมี edge หรือไม่ ก่อนเพิ่ม SL/TP และ risk management

3
Risk

รัน Agent + SL/TP

ถ้า Pure Agent พอใช้ได้ ให้ลอง Agent + SL/TP เพื่อดูผลใน setup ที่ใกล้ live มากขึ้น

4
Read

อ่านผลและ equity curve

ดู metrics ใน log, เปิด equity chart และตรวจว่า trades ไม่บางเกินไป ไม่ใช่กำไรจาก trade เดียว และ drawdown ไม่เกินรับได้

5
Gate

ตัดสินใจไปต่อหรือย้อนกลับ

ถ้าผ่าน Backtest ให้ไป Walk-Forward. ถ้าไม่ผ่าน ให้ย้อนกลับไป Quality Check, Regime Check, Training settings หรือ train ใหม่

ไฟล์ที่ Backtest ใช้และสร้างออกมา

ไฟล์ / ค่า หน้าที่ หมายเหตุ
<model>.zip โมเดลที่เลือกใน dropdown ต้องเป็นโมเดลที่ train สำเร็จแล้ว
<model>_norm.csv ใช้ normalize feature ตอน backtest ให้ scale ตรงกับ train ถ้าขาด ผล backtest อาจไม่น่าเชื่อถือ
Test Dataset CSV ข้อมูลที่ใช้ทดสอบโมเดล ควรเป็นช่วงอนาคต/ท้ายไฟล์ หรือชุด out-of-sample ที่ไม่ใช้ train
<model>_live_bt_trades.csv trade log ที่ backtest สร้าง ใช้ตรวจ trade รายตัวและใช้ generate chart ต่อ
<model>_live_bt_equity.png equity curve และ drawdown chart ใช้ดูว่ากำไรเรียบหรือได้จาก trade เดียว / drawdown หนักไหม

Backtest Mode: เลือกแบบไหน

Pure Agent

ตรงกับ environment ตอน train

mode นี้ไม่บังคับ SL/TP เพิ่มจากภายนอก จึงเหมาะสำหรับดูว่า agent ที่ train มาตัดสินใจเปิด/ปิดเองได้ดีแค่ไหน

ใช้เป็น baseline หลัง train รอบแรก

Agent + SL/TP

ใกล้ logic live มากขึ้น

mode นี้เพิ่ม ATR-based SL/TP, risk per trade, max positions, hard drawdown stop และ max hold logic

ใช้ดูความเหมาะสมก่อนคิดเรื่อง deploy/live broker

หลักการสอน: ใช้ Pure Agent เพื่อวัดคุณภาพ policy ที่ train มา แล้วใช้ Agent + SL/TP เพื่อดูว่าถ้าใส่ risk management แบบ live ผลยังอยู่รอดไหม

ค่าหลักในหน้า Backtest

ค่า ความหมาย คำแนะนำเริ่มต้น
Confidence Threshold ถ้า action Buy/Sell มีความมั่นใจต่ำกว่าค่านี้ ระบบจะ skip signal เริ่ม 0.85; Close ไม่ควรถูกกรองด้วย threshold
Skip % / Start fraction ข้ามช่วงต้นของ dataset แล้ว test เฉพาะท้ายไฟล์ เช่น 85 = test last 15% ใช้เมื่อต้องการทดสอบ out-of-sample จากไฟล์เดียว
Risk % ความเสี่ยงต่อ trade ใน mode live-realistic เริ่ม 0.01 หรือ 1%
Max Positions จำนวน position พร้อมกันสูงสุด เริ่ม 3; ถ้ามี skipped max positions เยอะให้ review strategy
ATR × SL / ATR × TP ระดับ stop loss / take profit จาก ATR proxy หน้า GUI ตั้งต้น 2.0 / 4.0 สำหรับ backtest live logic
Window Size ขนาด state window ปล่อย blank/auto เพราะ backtest auto-detect จาก observation space ของ model

อ่าน Metrics อย่างไร

Metric แปลว่าอะไร สัญญาณที่ควรระวัง
Return / Total P&L ผลตอบแทนรวมของ backtest กำไรแต่ drawdown หนักมาก หรือกำไรจาก trade เดียว
Profit Factor (PF) กำไรรวมจาก trade ชนะ / ขาดทุนรวมจาก trade แพ้ PF < 1 คือยังไม่ผ่าน, PF สูงมากแต่ trades น้อยต้องสงสัย
Win Rate สัดส่วน trade ที่ชนะ Win rate สูงแต่ avg loss ใหญ่มาก อาจยังไม่ดี
Max Drawdown การย่อตัวสูงสุดของ equity ถ้าลึกเกินรับได้ ห้ามข้ามไป deploy
Total Trades จำนวน trade ที่เกิดจริง น้อยเกินไปทำให้ผลไม่น่าเชื่อถือ แม้ PF จะดูดี
Skipped low confidence จำนวน Buy/Sell ที่ถูกกรองเพราะ confidence ต่ำ ถ้าเยอะมาก อาจ threshold สูงเกิน หรือ policy ไม่มั่นใจจริง
Exit reasons ปิดด้วย Close, SL, TP, max_hold, hard_stop หรือ end SL/max_hold/hard_stop เยอะผิดปกติ ต้อง review mode/risk/model

Decision: ผ่าน Backtest แล้วทำอะไรต่อ

สถานะ ความหมาย ตัดสินใจ
Go Return เป็นบวก, PF > 1, drawdown รับได้, trades มากพอ, equity curve ไม่พัง ไป Walk-Forward เพื่อตรวจ robustness หลายช่วงเวลา
Review ผลใกล้ผ่าน แต่ trades น้อย, confidence skip เยอะ, หรือ mode หนึ่งผ่านอีก mode ไม่ผ่าน ลองปรับ threshold/risk หรือกลับไป train อีก version แล้วเทียบ
Stop PF ต่ำกว่า 1, DD หนัก, no trades, หรือผลดีเฉพาะช่วงเดียว ห้าม export ใช้งานจริง ให้ย้อนกลับไป Quality Check/Train
ประโยคสอน user: Backtest คือด่านสอบของโมเดลหลัง train ถ้าโมเดลสอบตก อย่าแก้ด้วยการ export ไปลองจริง ให้กลับไปแก้ data, regime, feature หรือ training ก่อน
