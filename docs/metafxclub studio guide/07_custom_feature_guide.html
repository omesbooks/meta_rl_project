Custom Feature Guide

คู่มือนี้อธิบายวิธีออกแบบ feature สำหรับ dataset ของตัวเอง โดยยังให้ EA บน MT5 สร้าง feature ส่งเข้าโมเดลได้ตรงกับตอน train ตาม flow dataset → train → export → EA live

Workflow ที่ต้องรักษาไว้

ระบบนี้เปิดให้ปรับ feature ได้ แต่มีข้อแลกเปลี่ยนสำคัญ: feature ที่ใช้ตอน train ต้องเป็น feature ชุดเดียวกับที่ EA คำนวณตอน live ทั้งชื่อ, ลำดับ, จำนวน, ค่า parameter และวิธีคำนวณ

1

ออกแบบ feature และ parameter

กำหนดว่าจะใช้ indicator อะไร, period เท่าไร, ใช้ aggregate แบบไหน และชื่อ column อะไรใน CSV

2

ให้ DataCollector และ EA ใช้สูตรเดียวกัน

แก้ DataCollector_RL.mq5 สำหรับ input/sidecar และแก้ RL_Indicators.mqh สำหรับการคำนวณจริง

3

บันทึก parameter ลง .params.json

ไฟล์ sidecar นี้คือสัญญาว่า dataset ถูกสร้างด้วย settings อะไร และ export จะฝังค่านี้ลง config ของ EA

4

เก็บ CSV แล้ว import เข้าโปรเจค

ใช้ DataCollector_RL ใน MT5 แล้วนำ CSV + .params.json เข้า Data Tools เพื่อให้ training และ export เห็น config เดียวกัน

5

Train → Export → Test EA

Train โมเดลใหม่เสมอ แล้ว export ONNX/EA ใหม่ จากนั้นตรวจ log ว่า EA map feature ได้ครบ ไม่มี unknown feature

ห้ามใช้ model เก่ากับ feature format ใหม่ เพราะ PPO จำ pattern จาก feature distribution เดิมไว้แล้ว ถ้า column หรือสูตรเปลี่ยน ต้อง train/export ใหม่ทั้งชุด

Feature Contract

Name

ชื่อ feature ต้องตรง

ชื่อ column ใน CSV จะกลายเป็น RL_FEATURE_NAMES ตอน export และ EA จะหา feature ชื่อนั้นใน RL_ALL_FEATURES

Order

ลำดับต้องตรงกับ model

export ใช้ลำดับจาก norm stats ของโมเดล ถ้าเปลี่ยนลำดับเองหลัง train ค่า state ที่เข้า ONNX จะผิดทันที

Params

parameter ต้องฝังไปถึง EA

period, threshold, toggle และ custom settings ต้องอยู่ใน .params.json เพื่อให้ RL_ApplyDataCollectorConfig() ตั้งค่า EA ก่อนสร้าง handles

ไฟล์ หน้าที่ใน custom feature flow
mt5_files/MQL5/Experts/DataCollector_RL.mq5 รับ input จาก user, เรียก feature builder, เขียน CSV และเขียน .params.json
mt5_files/MQL5/Include/RL_Indicators.mqh เป็น source of truth ของ feature ที่ EA และ collector คำนวณร่วมกัน
rl_train.py อ่าน numeric feature columns จาก CSV, normalize ด้วย train stats, forward sidecar ไป model artifacts
export_to_onnx.py เขียน RL_FEATURE_NAMES, mean/std และฝัง params เป็น RL_ApplyDataCollectorConfig()
ML_RL_Trader_template.mq5 EA เรียก config ก่อน init indicators แล้วคำนวณ feature ส่งเข้า ONNX

ปรับ Period Range ด้วย Min / Max / Step

feature aggregate เช่น rsi_min, rsi_max, rsi_mean, rsi_std ไม่ได้สร้าง column แยกทุก period แต่สร้างสถิติรวมจากชุด period ที่กำหนด

Example

RSI 4 ถึง 30 step 2

ถ้าตั้ง InpRSI_Pmin = 4, InpRSI_Pmax = 30, InpRSI_Pstep = 2 ระบบจะใช้ period: 

4, 6, 8, 10, 12, ..., 30

แล้วคำนวณออกมาเป็น rsi_min/max/mean/std เหมือนเดิม

Important

จำนวน column อาจเท่าเดิม แต่ distribution เปลี่ยน

แม้ column ยังชื่อเดิม แต่ค่า feature เปลี่ยนตามชุด period จึงต้อง train model ใหม่ และ export EA ใหม่เสมอ

ค่า step จะถูกเขียนลง .params.json เช่น RSI_PSTEP, ATR_PSTEP, STOCH_PSTEP, CCI_PSTEP, WPR_PSTEP, ADX_PSTEP แล้ว export จะฝังค่าเหล่านี้ลง config ของ EA

เพิ่ม Indicator หรือ Feature ใหม่

Python training รับ numeric column ใหม่ได้ แต่ EA จะส่ง feature ได้ถูกต้องก็ต่อเมื่อ RL_Indicators.mqh รู้จักและคำนวณ feature นั้นได้ด้วย

1

ตั้งชื่อ feature แบบนิ่ง

ใช้ชื่อสั้น อ่านง่าย เป็น snake_case เช่น custom_momentum_20 หรือ adx_slope_14

2

เพิ่มชื่อใน RL_ALL_FEATURES

เพิ่มชื่อท้าย master list และเพิ่ม RL_ALL_FEATURES_COUNT ให้ตรงจำนวนจริง

3

เพิ่ม handle / buffer / สูตรคำนวณ

สร้าง indicator handle ใน RL_InitIndicators(), release ใน RL_DeinitIndicators(), แล้วเขียนค่าใน RL_BuildAllFeatures()

4

เพิ่ม parameter ลง .params.json

ถ้า feature มี period หรือ threshold ให้ DataCollector เขียน key ลง sidecar เพื่อให้ export ฝังไป EA

5

เก็บ dataset และ train ใหม่

หลัง feature set เปลี่ยน ให้สร้าง CSV ใหม่, import ใหม่, train ใหม่, backtest ใหม่ และ export EA ใหม่

ถ้า train CSV มี column ที่ EA ไม่รู้จัก ตอน export จะยังเขียนชื่อ feature นั้นลง config ได้ แต่ตอน EA รันจะขึ้น unknown feature และใช้ค่า 0.0 แทน ทำให้ inference ไม่น่าเชื่อถือ

.params.json คือสะพานระหว่าง Dataset กับ EA

sidecar นี้ควรอยู่คู่กับ CSV เสมอ เช่น gu_h4_dataset.csv คู่กับ gu_h4_dataset.params.json 

{
  "RSI_PMIN": 4,
  "RSI_PMAX": 30,
  "RSI_PSTEP": 2,
  "ATR_PMIN": 5,
  "ATR_PMAX": 50,
  "ATR_PSTEP": 1,
  "CUSTOM_FEATURE_VERSION": 1
}

ตอน train, rl_train.py จะ copy sidecar ไปไว้กับ model artifacts และตอน export, export_to_onnx.py จะอ่าน sidecar นี้เพื่อ generate RL_ApplyDataCollectorConfig() ในไฟล์ config ของ EA

ถ้าต้องการให้ user ทดลอง feature หลายชุด ให้ตั้งชื่อ dataset/model ให้บอก recipe ชัดเจน เช่น gu_h4_rsi_step2_v1.csv และ rl_gu_h4_rsi_step2_v1

Checklist ก่อนเชื่อผล Backtest / EA

Dataset

ตรวจ CSV

  • timestamp เรียงเวลาและไม่มี duplicate ที่ผิดปกติ
  • feature เป็น numeric และไม่มีค่า NaN/inf ค้าง
  • ไม่มี future/target/leaky column ปนเป็น feature
  • มี .params.json อยู่ข้าง CSV
EA

ตรวจ MT5 log

  • ไม่มี unknown feature ใน RL_BuildFeatureMap()
  • feature count ตรงกับ config ของโมเดล
  • debug log แสดง feature/state ได้ ไม่เป็นศูนย์ทั้งชุด
  • Strategy Tester รันบน symbol/timeframe เดียวกับ dataset
Backtest ใน Python ผ่าน ไม่ได้แปลว่า EA live ถูกต้องเสมอไป ต้องตรวจ parity ใน MT5 ด้วย โดยเฉพาะหลังเพิ่ม custom indicator หรือเปลี่ยน parameter ของ collector

สิ่งที่ User ปรับได้ และสิ่งที่ต้องระวัง

ปรับได้ เงื่อนไข ต้องทำต่อ
Period min/max/step ต้องเขียนลง .params.json และ EA ต้องใช้ค่าเดียวกัน เก็บ CSV ใหม่, train ใหม่, export ใหม่
Candle pattern toggles/thresholds ต้องส่งผ่าน sidecar และ RL_ApplyDataCollectorConfig() ตรวจว่า CandlePatterns compile แล้ว
เพิ่ม indicator ใหม่ ต้องเพิ่มทั้ง master feature list และสูตรใน RL_Indicators.mqh ตรวจ unknown feature = 0 ก่อนรันจริง
ลบ feature ด้วย Clean Redundant Features ลบจาก CSV train/test ได้ แต่ EA ยังต้องรู้จัก feature ที่ model เหลือใช้อยู่ train/export จาก CSV ที่ clean แล้วเท่านั้น

Prompt สำหรับสั่ง Codex เพิ่ม Custom Feature

ใช้ prompt นี้เมื่อ user มีไอเดีย indicator หรือ feature ของตัวเอง และต้องการให้ Codex แก้ครบทั้ง flow ไม่ใช่เพิ่มแค่ column ใน CSV

ก่อนส่ง prompt ให้แทนค่าช่อง <...> ให้ชัดเจน เช่นชื่อ feature, สูตรคำนวณ, period/lookback และ parameter ที่อยากให้ปรับได้ใน MT5 
อยู่ในโปรเจค C:\Users\omesb\Documents\claude code\pycaret_trainer

ช่วยเพิ่ม custom feature ใหม่เข้า flow RL ให้ครบ dataset → train → export → EA live

Feature ที่ต้องการ:
- ชื่อ feature/column: <ใส่ชื่อ เช่น ma_slope_50>
- สูตรคำนวณ: <อธิบายสูตร เช่น slope ของ EMA50 เทียบกับ 10 bars ก่อนหน้า>
- input/parameter ที่ user ควรปรับได้ใน MT5: <เช่น InpMASlope_Period=50, InpMASlope_Lookback=10>
- ต้องการให้เขียนลง .params.json ไหม: ใช่
- feature นี้ใช้ closed bar เท่านั้น ห้ามใช้ future data
