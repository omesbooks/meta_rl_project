Flow Step 1 · Data Prep

เตรียมข้อมูลจาก MT5 เข้า Dashboard

บทนี้อธิบายช่วงแรกสุดของ workflow: ให้ DataCollector_RL ใน MT5 สร้างไฟล์ข้อมูล, ตรวจว่าไฟล์ถูกบันทึกจริง, แล้ว import CSV พร้อม .params.json เข้ามาในโปรเจกต์เพื่อใช้ train RL ต่อ

ภาพรวม ไฟล์ที่ต้องมี ขั้นตอนใน Dashboard เช็คว่า Data ใช้ได้ ปัญหาที่พบบ่อย ส่งต่อขั้นถัดไป

ภาพรวมแบบเข้าใจง่าย

1 · Collect

MT5 เป็นคนสร้างข้อมูล

เปิด DataCollector_RL ใน MT5 เพื่อ export bars และ feature ที่ EA ฝั่ง MT5 ใช้จริง ผลลัพธ์หลักคือไฟล์ .csv

2 · Import

Dashboard คัดลอกเข้าโปรเจกต์

หน้า Data Tools จะอ่านไฟล์จาก Common\Files แล้วคัดลอกมาไว้ที่ root ของโปรเจกต์ เช่น uj_h4_dataset.csv

3 · Check

เช็คก่อน train

ก่อนเอาไป train ต้องเช็คว่า CSV มี rows จริง, timestamp เรียงได้, feature ไม่ว่าง และถ้าจะ deploy ต้องมี .params.json คู่กัน

จำสั้น ๆ: ขั้นนี้ยังไม่ใช่การ train และไม่ใช่การสร้าง target UP/DOWN/FLAT สำหรับ RL เราต้องการ market state ที่สะอาดและ replay ได้เหมือนฝั่ง MT5 มากที่สุด

ไฟล์ที่ต้องเข้าใจ

ไฟล์ / โฟลเดอร์ ทำหน้าที่อะไร สำคัญกับ RL อย่างไร
Common\Files โฟลเดอร์กลางของ MetaTrader ที่ EA/Script สามารถเขียนไฟล์ออกมาได้ ปกติอยู่ที่
C:\Users\<user>\AppData\Roaming\MetaQuotes\Terminal\Common\Files
 เป็นจุดรับไฟล์จาก DataCollector_RL ก่อน dashboard จะ import เข้ามา
uj_h4_dataset.csv ไฟล์ข้อมูลแท่งราคาและ feature เช่น timestamp, OHLCV, indicators, candle features เป็น dataset หลักที่ rl_train.py ใช้สร้าง environment ให้ agent ฝึก
uj_h4_dataset.params.json sidecar config ที่บอกค่า parameter ของ collector เช่น indicator periods และ setting ที่ใช้ตอนเก็บข้อมูล ช่วยให้ตอน export/deploy ฝั่ง EA คำนวณ feature ให้ตรงกับตอน train ลดความเสี่ยง train อย่างหนึ่งแต่ live อีกอย่างหนึ่ง
training_data_*.csv ชื่อไฟล์ปลายทางใน project folder หลัง import เป็นไฟล์ที่ user เลือกต่อในหน้า Train, Pipeline, Backtest หรือ Data Tools module อื่น

ขั้นตอนใช้งานใน Dashboard

1
MT5

รัน DataCollector_RL ใน MT5

ให้เปิด chart symbol/timeframe ที่ต้องการ แล้วรัน collector เพื่อให้สร้าง CSV ลงใน Common\Files ตัวอย่างชื่อไฟล์เช่น uj_h4_dataset.csv

2
Data Tools

เปิดหน้า Import from DataCollector_RL

ใน Dashboard ไปที่ Data Tools แล้วดู card Import from DataCollector_RL dropdown Source CSV จะอ่านไฟล์จาก Common\Files

  • ถ้าไม่เห็นไฟล์ ให้กด Refresh
  • ถ้ายังไม่เห็น ให้กลับไปเช็คว่า MT5 เขียนไฟล์ไว้ใน Common Files จริงหรือไม่
3
Name

ตั้ง Output basename

ตั้งชื่อไฟล์ปลายทางให้อ่านแล้วรู้ทันทีว่าเป็น symbol/timeframe/ชุดข้อมูลอะไร เช่น training_data_uj_h4_dataset.csv

  • ใช้ชื่อสั้นแต่สื่อความหมาย เช่น training_data_xau_h1_2026.csv
  • หลีกเลี่ยงชื่อซ้ำกับไฟล์ที่สำคัญ เพราะ import จะเขียนทับไฟล์ปลายทางได้
4
Import

กด Import CSV + params

Dashboard จะคัดลอก CSV เข้ามาที่ project folder และถ้ามีไฟล์ .params.json ชื่อเดียวกัน ก็จะคัดลอกมาคู่กันด้วย

หลัง import สำเร็จ dropdown ของหน้า Data Tools ควรเห็นไฟล์ใหม่ทันที ถ้าเป็นหน้าที่เปิดค้างไว้ ให้ใช้ปุ่ม Refresh ของ module นั้นได้

Checklist: เช็คว่า CSV มีข้อมูลจริงและพร้อมใช้

ต้องผ่าน

เช็คระดับไฟล์

  • มีไฟล์ .csv ใน project folder
  • ขนาดไฟล์ไม่ใช่ 0 KB
  • มีไฟล์ .params.json คู่กัน ถ้าจะ export/deploy ไป MT5
  • ชื่อไฟล์ไม่มีช่องว่างหรืออักขระแปลกที่ทำให้เลือกยาก
ต้องผ่าน

เช็คระดับข้อมูล

  • มี column เวลา เช่น timestamp
  • มีราคา open, high, low, close
  • timestamp เรียงตามเวลาได้ และไม่มีช่วงเวลาย้อนกลับผิดปกติ
  • มีจำนวน rows มากพอสำหรับ train/test split
  • feature หลักไม่เป็นค่าว่างทั้ง column
สิ่งที่เห็น แปลว่าอะไร ควรทำต่อ
CSV มี rows เยอะและมี params คู่กัน ข้อมูลพร้อมเข้าสู่ขั้น quality check หรือ train ได้ ไปหน้า Data Tools ต่อเพื่อ split/feature analysis หรือไปหน้า Train
CSV มีแต่ไม่มี params ยัง train ได้ แต่ parity ตอน export/deploy อาจไม่ครบ ใช้ได้สำหรับทดลอง แต่ก่อน deploy ควรหา params มาคู่กัน
CSV ไม่โผล่ใน dropdown dashboard ยังไม่ rescan หรือไฟล์ไม่ได้อยู่ใน path ที่คาดไว้ กด Refresh, เช็ค Common Files, หรือเปิดโฟลเดอร์ project ดูว่าไฟล์อยู่จริง
CSV เปิดแล้ว column ผิดหรือ timestamp อ่านไม่ได้ collector/export format อาจไม่ตรงกับที่ dashboard คาดไว้ กลับไปเช็ค DataCollector_RL และ format ของไฟล์ก่อน train

ปัญหาที่พบบ่อย

ไม่เห็นไฟล์

Source CSV เป็น none

มักเกิดจาก DataCollector_RL ยังไม่ได้เขียนไฟล์ลง Common\Files หรือ dashboard เปิดค้างก่อนมีไฟล์ ให้กด Refresh ใน card Import from DataCollector_RL

ไม่มี params

CSV มาแต่ sidecar ไม่มา

ถ้าไม่มี .params.json ระบบจะใช้ default ฝั่ง MT5 ตอน export ได้บางกรณี แต่ควรถือว่ายังไม่พร้อม production เพราะ feature parity อาจคลาดเคลื่อน

ชื่อไฟล์สับสน

ไม่รู้ว่าไฟล์ไหนล่าสุด

ตั้งชื่อ output ให้มี symbol/timeframe/date หรือเวอร์ชัน เช่น training_data_uj_h4_2026_06.csv

เข้าใจผิด

Import ไม่ได้แปลว่า Train แล้ว

ปุ่ม Import ทำหน้าที่คัดลอกไฟล์เท่านั้น ไม่ได้รัน train, ไม่ได้ backtest และไม่ได้รับประกันว่า model จะดี

สำหรับ RL: ไม่จำเป็นต้องสร้าง future_return หรือ target UP/DOWN/FLAT ในขั้นนี้ เพราะ RL เรียนจาก state, action, reward ไม่ใช่ supervised label แบบ PyCaret classification

ส่งต่อขั้นถัดไป

เมื่อ import เสร็จและเช็คว่า CSV ใช้ได้แล้ว ให้ไปต่อได้ 2 ทาง:

ทางแนะนำ

Data Quality ก่อน Train

เปิดหน้า Data Tools ต่อเพื่อ split train/test by date และใช้ Feature Analysis ตรวจ correlation ก่อน train จริง เหมาะกับงานสอน user เพราะเห็นวินัยข้อมูลชัดเจน

อ่านต่อ: Data Tools Modules

ทางเร็ว

ไป Train โดยตรง

ถ้าเป็นไฟล์ที่เชื่อถือได้และมีข้อมูลครบ สามารถเลือก CSV ในหน้า Train แล้วเริ่ม train model ได้ แต่ยังต้อง backtest/walk-forward ก่อน export
