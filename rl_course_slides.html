💡 Concepts ที่ต้องรู้ก่อน

คำศัพท์สำคัญที่จะเจอตลอดคอร์ส — รู้ไว้ก่อนช่วยให้เข้าใจง่ายขึ้น

🎯 ML (Machine Learning)

คอมพิวเตอร์เรียนรู้จากข้อมูล โดยไม่ต้องเขียนกฎทุกอย่างเอง

ตัวอย่าง: ดูประวัติราคา → ทาย UP/DOWN

🤖 RL (Reinforcement Learning)

ML รูปแบบหนึ่ง — Agent ลองทำ + ได้ reward → เรียนเก่งขึ้น

ตัวอย่าง: สอนสุนัขด้วยขนม

🧠 Agent

"สมอง" ที่ตัดสินใจ — ในนี้คือ Neural Network

มันคือคนที่ตัดสินใจซื้อ/ขาย

🌍 Environment

"โลก" ที่ Agent โต้ตอบ — ในนี้คือตลาดจำลอง

ส่ง state + reward ให้ agent

🎁 Reward

คะแนนที่ Agent ได้ — บอกว่าทำดีหรือแย่

ในนี้: P&L ของ trade

📐 Features

"ตัวแปร" ที่ป้อนเข้า model — RSI, EMA, ATR ฯลฯ

ยิ่งดี → model ฉลาดขึ้น

⚖️ Step 1: Relabel — แก้ Class Imbalance

❌ ก่อน Relabel

FLAT : 73% ████████████████████
DOWN : 14% ████
UP : 13% ████

→ Model ทาย FLAT ทุกครั้ง = "Lazy Classifier"

✅ หลัง Relabel (Quantile 33/33/33)

FLAT : 33% ███████████
DOWN : 33% ███████████

→ Model ต้องเรียนรู้จริง

Relabel: เปลี่ยน threshold จาก fixed (เช่น ±0.2%) เป็น quantile-based (33/33/33) → balance อัตโนมัติ

🤖 PPO คืออะไร? (อธิบายง่ายๆ)

PPO = Proximal Policy Optimization — Algorithm สอน Neural Network ให้ตัดสินใจดีขึ้น

🎯 ทำงานยังไง

  1. Agent ดู state → ตัดสินใจ (Buy/Sell/Hold)
  2. ลงมือทำ → เห็น reward (+/-)
  3. ปรับ NN: action ดี → เพิ่ม prob, แย่ → ลด prob
  4. ทำซ้ำ 200,000 ครั้ง → เก่งขึ้น

💡 Proximal = "ใกล้ๆ"

ห้าม policy เปลี่ยนเร็วเกิน ±20% ต่อ update

เปรียบเทียบ:

❌ ก่อน PPO: Update เร็ว → ลืมของเก่า
✅ PPO: Clip 0.2 → ค่อยๆ ปรับ stable

เปรียบเหมือน: สอนสุนัขเฝ้าบ้าน — ค่อยๆ ฝึก ไม่ใช่เปลี่ยนวิธีทุกวัน
PPO = "Standard" ของ RL Trading ใน 2024+ (default ของ stable-baselines3)

🧠 Step 2: RL Training (PPO)

🌍 TradingEnv (Gymnasium)

  • State: 35 features × 10 bars + position
  • Action: 0=Hold, 1=Buy, 2=Sell, 3=Close
  • Reward: realized P&L only at close

🎯 PPO Configuration

  • learning_rate: 3e-4
  • n_steps: 2048 (rollout)
  • 200,000 total steps (~30 นาที CPU)
  • Network: [128, 64] hidden layers
⚠️ Reward Shaping = หัวใจของ RL: ใช้ realized reward + max_hold_bars=30
กัน "Buy-and-Hold trap" ที่ agent เปิด long ครั้งเดียวถือ 3 ปี!

🛡️ Risk Management — 3 Layers

🔴 L1
Hard Stop (Critical)
Equity < 85% of initial
→ ปิดทุก position + หยุดเทรด · manual restart
🟡 L2
Soft Pause (Warning)
Rolling PF < 0.9 (50 trades)
→ หยุดเปิด trade ใหม่ 24 ชม · trigger fine-tune
🟢 L3
Position Sizing (Always)
Risk 1% per trade · Max 3 positions
→ Dynamic lot ตาม SL distance + ATR

💧 Concept Drift — ทำไม Model เก่าใช้ไม่ได้นาน?

Model เทรนวันนี้ดี ปีหน้าอาจไม่ได้แล้ว — เพราะ ตลาดเปลี่ยน

📉 ทำไมเกิด Drift?

  • Algorithm trader เพิ่ม → arbitrage หายไป
  • นโยบายเศรษฐกิจเปลี่ยน (FED, ECB)
  • เหตุการณ์ใหม่ (COVID, สงคราม, AI boom)
  • Liquidity เปลี่ยน
  • Pattern ที่เคย work → คนใช้กันหมด → ไม่ work แล้ว

✅ ทางแก้

  • ตรวจจับ: Walk-forward validation
  • วัดผล live: rolling PF (last 50 trades)
  • Update: Fine-tune ทุก 3 เดือน
  • Reset: Retrain scratch ทุก 1 ปี
"ตลาดไม่เคยหยุดเปลี่ยน → Model ก็ต้องไม่หยุดเรียน"
Edge ที่ได้มา = ต้องดูแลรักษา ไม่ใช่ปลูกครั้งเดียวแล้วทิ้ง

🎯 Mindset สำคัญ

✅ DO

  • เริ่มเล็ก, validate ทุกขั้นตอน
  • ใช้ demo ก่อน live เสมอ
  • Walk-forward ก่อน deploy ทุกครั้ง
  • Backup model เก่าก่อน update
  • Risk management = ห้ามข้าม
  • Log ทุก trade + state
  • ยอมรับว่า edge มีวันหมด

❌ DON'T

  • ห้าม deploy โดยไม่ผ่าน WF
  • ห้าม auto-deploy โดยไม่มี gate
  • ห้าม fine-tune ถี่เกิน (cooldown 14d)
  • ห้ามใช้ leverage สูงตอนเริ่ม
  • ห้ามคาดหวังกำไร 100%
  • ห้าม fine-tune บน new only (forget!)
  • ห้าม ignore drift signals

🚨 5 ข้อผิดพลาดที่คนทำบ่อย

เจอกันบ่อย — ระวังไว้จะประหยัดเวลาได้เยอะ

1️⃣ Data Leakage

อาการ: Win rate 100% ดูเทพมาก

สาเหตุ: future_return เผลอเป็น feature → model "เห็นอนาคต"

✅ Drop leaky cols ก่อนเทรน

2️⃣ Class Imbalance

อาการ: Total trades = 0 หรือทำเฉพาะ class เดียว

สาเหตุ: FLAT 73% → model ขี้เกียจ

✅ Relabel quantile 33/33/33

3️⃣ Buy-and-Hold Trap

อาการ: RL agent เปิด long ครั้งเดียว ถือ 3 ปี

สาเหตุ: MTM reward ทำให้ "ถือเฉยๆ" ก็ได้ reward

✅ Realized reward + max_hold_bars

4️⃣ Skip Walk-Forward

อาการ: Backtest ดี → live พังหมด

สาเหตุ: Single split หลอก, มี overfit ที่ซ่อนอยู่

✅ Walk-forward 5 windows ทุกครั้งก่อน deploy

5️⃣ Forgot Maintenance

อาการ: Deploy แล้วทิ้ง → กำไรหายไปเรื่อยๆ

สาเหตุ: Concept drift — ตลาดเปลี่ยน

✅ Fine-tune ทุก 3 เดือน

💡 จำไว้

คนเก่งไม่ใช่ ไม่เคยทำผิด แต่เป็นคนที่ ตรวจเจอเร็ว + แก้ไขได้

📖 Glossary — คำศัพท์สำคัญ

คำศัพท์ที่ใช้ตลอดคอร์ส — เก็บไว้อ้างอิง

📊 Backtest

ทดสอบ strategy บนข้อมูลในอดีต — ดูว่าถ้าเทรดจริงตอนนั้น ได้กำไรแค่ไหน

⚖️ Walk-Forward

Backtest แบบเลื่อนหน้าต่างทดสอบ → พิสูจน์ว่า model robust ทุกช่วง

📐 Profit Factor (PF)

กำไรรวม ÷ ขาดทุนรวม → > 1.0 = กำไร, > 1.5 = ดีมาก

📉 Drawdown (DD)

% ที่ equity ตกจากจุดสูงสุด — ยิ่งน้อยยิ่งดี (< 20%)

🎯 Win Rate

% ของ trade ที่กำไร — > 50% มี edge, แต่ไม่ใช่ทุกอย่าง

📊 Sharpe Ratio

Return ÷ Volatility → วัด risk-adjusted return (> 1.0 = ดี)

🔄 Fine-tune

Train ต่อจาก model เก่า — ใช้ data ใหม่ ปรับให้ทันตลาด

💧 Concept Drift

ตลาดเปลี่ยน → model เก่าใช้ไม่ได้แล้ว → ต้อง update

🌍 Environment (Env)

"โลก" ที่ agent อยู่ → ส่ง state + reward ให้ ใน RL

🧠 Policy

"กฎการตัดสินใจ" ของ agent — ดู state แล้ว choose action ยังไง

📊 State

"ข้อมูลที่ agent เห็น" ในเวลานี้ — features + position + P&L

⚡ Action

สิ่งที่ agent ทำได้ — Hold / Buy / Sell / Close
